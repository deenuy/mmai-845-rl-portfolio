# Roadmap Deviations Log

## Purpose

This log records any development behavior, implementation decision, phase scope change, or validation rule that differs from the approved roadmap.

Use this file as the single source of truth for later summary, governance review, and final reporting.

## Logging Rules

- Add one entry for each meaningful deviation.
- Log the deviation as soon as the change is accepted.
- Keep entries factual and concise.
- Reference the affected roadmap phase.
- Document the reason, expected impact, and approval status.
